# 稳定裕度：从伯德图到闭环鲁棒性

## 核心结论

稳定裕度不是“系统已经稳定”的同义词，而是系统距离临界失稳还有多远。工程上通常用增益裕度和相位裕度衡量：增益裕度回答增益还能放大多少，相位裕度回答额外延迟或相位滞后还能增加多少。裕度越大，系统对参数误差、未建模动态和噪声通常越不敏感，但响应也可能更慢。

## 基本定义

对开环传递函数 L(s)，在相位穿越 -180° 的频率处读取幅值。如果幅值低于 0 dB，系统还可以增加增益；从当前幅值到 0 dB 的差值就是增益裕度。对幅值穿越 0 dB 的频率，读取当前相位与 -180° 的距离，就是相位裕度。计算时必须明确使用开环信号、反馈符号和频率单位；把闭环传递函数直接拿来读裕度是常见错误。

## 伯德图判读流程

先画幅频和相频曲线，标出 0 dB 交越频率与 -180° 相位交越频率；再分别读取另一条曲线的值；最后将 dB、倍数、角度和符号统一。多个交越频率时不能只看第一个，应检查所有可能导致最小裕度的交越点。右半平面极点或零点、纯延迟、采样和非最小相位环节会使相位裕度的直觉判断失效，必要时使用 Nyquist 图辅助确认。

## 工程解释

相位裕度小，意味着系统只需增加少量延迟、滤波器相位滞后或模型误差就可能出现振荡。增益裕度小，意味着执行器增益、传感器标定或参数变化可能把闭环推向不稳定。裕度过大也不是绝对最优：为了获得大裕度而把带宽压得过低，会牺牲跟踪速度和扰动抑制。设计目标应在稳定性、速度、噪声和鲁棒性之间折中。

## 调整方法

增加超前校正通常提高相位裕度并改善响应速度；滞后校正有利于提高低频增益和稳态精度，但会引入额外相位滞后；降低交越频率可以避开高频未建模动态和噪声，但会变慢；陷波器可压低结构共振峰，必须检查其相位影响。每次调整后都要重新读取两个裕度、交越频率、闭环带宽和时域超调，不能只看伯德图某一点。

## 实际设计流程

1. 明确被控对象、传感器、执行器、反馈方向和工作点，得到可用于频域分析的开环模型。
2. 先检查开环极点、零点、纯延迟和非最小相位因素，再绘制幅频和相频曲线。
3. 找出全部幅值穿越频率和相位穿越频率，记录最小增益裕度、最小相位裕度及其对应频率。
4. 根据目标选择调整方向：需要提高速度时考虑超前校正，需要提高低频精度时考虑滞后校正，需要避开共振时降低交越频率或使用陷波器。
5. 每次修改后同时做频域和时域验证，检查闭环极点、阶跃超调、调节时间、稳态误差、噪声放大以及参数扰动下的裕度变化。

## 误区与边界

正裕度只说明在线性模型和当前工作点下距离临界条件还有余量，不等于实际系统在饱和、摩擦、死区、采样延迟和时变参数下必然稳定。多个交越点时应取最不利裕度；右半平面极点或零点存在时，伯德图读数必须结合 Nyquist 绕行方向解释。裕度过大也可能意味着带宽过低、跟踪变慢或扰动抑制不足，因此设计目标应同时考虑鲁棒性和性能。

## 小型判读示例

假设某开环系统在 0 dB 交越频率处的相位为 -135°，则相位裕度为 45°；若在 -180° 相位交越处的幅值为 -8 dB，则增益裕度为 8 dB。这个读数只能说明线性频域模型的相对余量，不能直接推出阶跃响应一定满足超调要求。应进一步观察闭环极点、时域响应和参数扰动后的裕度变化。

## 自测题

为什么要同时查看增益裕度和相位裕度？因为增益变化和相位滞后是两种不同的失稳路径。为什么滤波器可能降低相位裕度？因为滤波器在交越频率附近引入额外相位滞后。为什么裕度不是越大越好？因为过度降低交越频率会牺牲带宽、跟踪速度和扰动抑制。多个交越点时怎么办？应检查全部交越点并采用最不利结果。

## 调参后的复核顺序

调参后先确认开环增益没有因单位或反馈符号错误而整体平移，再确认交越频率落在模型可信范围内。随后比较调参前后的相位裕度、增益裕度和带宽，检查是否以牺牲一个指标换取另一个指标。最后做阶跃、正弦扫频、扰动抑制和参数蒙特卡洛测试；如果频域裕度改善但时域出现饱和或长时间振荡，应优先检查非线性和执行器限制，而不是继续增加补偿增益。

## 结论

稳定裕度应与带宽、时域响应和参数鲁棒性共同评估。

## 实践检查表

建立开环模型时，先确认反馈方向、信号单位和传递函数分子分母；绘制伯德图时标出所有 0 dB 与 -180° 交越，不要只读取第一个交点。每次读取都记录交越频率、幅值、相位、增益裕度和相位裕度，并注明采用的符号约定。修改控制器后，除了重新读裕度，还要检查闭环极点、阶跃响应超调、调节时间、稳态误差和噪声放大。若系统包含纯延迟、右半平面零点、采样环节、结构共振或未建模高频动态，应把它们显式纳入模型，再判断裕度是否足够。

## 典型判断题

相位裕度变小通常意味着相位滞后增加，系统更接近振荡；增益裕度变小通常意味着环路增益变化更容易造成失稳。增加带宽不等于稳定性变好，因为交越频率可能进入共振或未建模动态区域。增加滤波器阶数可能降低噪声，却也会消耗相位裕度。一个系统即使裕度为正，也可能因非线性、饱和、时变参数或执行器限制而出现实际振荡，因此还需要时域仿真、参数扫描和实验验证。
