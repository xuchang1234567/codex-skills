# MIT 计算机导论：课程练习与复习计划

## 使用方式

每完成一组视频，先关闭字幕独立完成题目，再回看对应分 P。每道题至少留下：思路、伪代码、正确性依据、复杂度、一个失败反例和测试结果。

## 第一组：算法与线性结构

### 题目 1：动态数组

实现支持 `append`、按索引访问、末尾删除、任意位置插入和删除的动态数组。容量不足时扩容，记录元素搬移次数。

验收：解释为什么追加的摊销复杂度可以较低，为什么头部插入仍然需要搬移大量元素；测试空数组、连续扩容和删除到空。

### 题目 2：哈希表

实现键值查找、插入、更新和删除，任选链式或开放寻址处理冲突。

验收：构造多个冲突键；测试扩容和删除；解释负载因子、平均复杂度和最坏情况；说明为什么哈希表不适合直接做范围查询。

### 题目 3：线性排序

分别实现计数排序或基数排序，并与比较排序对照。

验收：写清键范围、稳定性和额外空间前提；测试键范围远大于 n 的情况，解释为什么“线性”不一定更快。

## 第二组：树、堆与 BFS

### 题目 4：二叉树遍历

实现前序、中序、后序和层序遍历，分别提供递归版和迭代版。

验收：测试空树、单节点、只有左子树、只有右子树和完全树；画出调用栈或显式栈变化；说明遍历复杂度。

### 题目 5：最小堆

实现插入、查看最小值、删除最小值和建堆。

验收：逐步画出向上/向下调整；比较逐个插入建堆和批量建堆；说明堆能保证什么，不能保证什么。

### 题目 6：无权图 BFS

用邻接表实现 BFS，输出访问顺序、距离和从起点到目标的路径。

验收：测试重复边、环、不可达目标和非连通图；证明第一次发现节点时得到的是最少边数距离。

## 第三组：DFS 与最短路

### 题目 7：DFS 图分析

实现 DFS 的递归和显式栈版本，记录父节点、进入时间和离开时间。

验收：分别测试有向图和无向图；构造环；解释为什么无向图遇到父节点不能直接判定为环。

### 题目 8：Bellman-Ford

实现单源最短路和负权环检测，输出距离与父节点。

验收：测试正权图、含负权边图、不可达节点和负权环；手算一轮松弛并与程序日志对照。

### 题目 9：Dijkstra

用最小堆实现非负权图单源最短路，处理优先队列中的过期条目。

验收：测试多条等长路径、重复入队、不可达节点；加入一条负权边验证算法前提失效，并解释失败原因。

### 题目 10：算法选择器

给定一张图和权重条件，让程序或人工规则选择 BFS、Dijkstra 或 Bellman-Ford。

验收：规则必须检查无权、非负权、负权和负环风险；不能只根据图大小选择算法。

## 第四组：动态规划与复杂度

### 题目 11：递归到动态规划

任选一个有重叠子问题的递归问题，先写朴素递归，再写记忆化和自底向上版本。

验收：画递归树，标出重复子问题；写出状态完整语义；说明状态数量、转移数量和空间成本。

### 题目 12：路径恢复

在动态规划中保存决策或前驱，使程序不仅返回最优值，还返回具体方案。

验收：测试多个最优解、无解和最小输入；说明相同最优值时如何选择方案。

### 题目 13：复杂度审计

选取前面实现的 3 个算法，逐行估算主要操作成本，分别写最好、平均和最坏情况。

验收：不能只写一个 O 记号；要说明哈希假设、图的顶点/边数量、堆操作次数或 DP 状态数的来源。

## 综合项目：算法工具箱

实现一个小型算法工具箱，至少包括：动态数组、哈希表、堆、BFS、DFS、Dijkstra、Bellman-Ford 和一个动态规划问题。每个模块统一提供：输入格式、输出格式、错误处理、复杂度说明和测试样例。

项目验收：

- 每个算法有至少一个正常样例和一个反例；
- 图算法可以恢复路径；
- 最短路模块会检查边权前提；
- DP 模块能输出状态表或关键决策；
- README 中列出选择算法的判断流程；
- 测试覆盖空输入、单节点、重复边、不可达和极端规模。

## 四周复习计划

### 第 1 周

完成动态数组、哈希和排序，实现并记录复杂度。重点理解摊销、冲突和输入前提。

### 第 2 周

完成树、堆和 BFS。重点练习递归边界、堆调整和路径恢复。

### 第 3 周

完成 DFS、Bellman-Ford、Dijkstra。重点比较算法前提、松弛和负权反例。

### 第 4 周

完成一个动态规划问题和综合工具箱，整理错题、复杂度审计和算法选择表。

## 最终自评标准

- 能从题目约束判断候选算法，而不是凭熟悉程度选算法。
- 能写出状态定义、循环不变量或递归正确性说明。
- 能解释算法为什么在某些输入下失效。
- 能区分平均、最坏、摊销和渐进复杂度。
- 能用测试和反例发现代码问题。
- 能把算法实现、复杂度和工程限制放在同一个设计决策中。
