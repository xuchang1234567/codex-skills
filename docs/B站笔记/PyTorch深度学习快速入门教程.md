## 一、环境与工具

安装时确认 Python、PyTorch 和 CUDA 组合是否匹配。没有 NVIDIA 显卡时，torch.cuda.is_available() 返回 False 是正常现象，不代表 PyTorch 安装失败；可以使用 CPU 继续学习，只是训练速度可能更慢。[1]

环境问题可分三类：Python 或包导入失败，检查解释器和虚拟环境；CUDA 不可用，检查显卡、驱动和 PyTorch 构建版本；代码能运行但速度慢，检查设备选择、数据加载和模型是否真的迁移到 GPU。

PyCharm 适合多文件项目、断点调试和长期维护；Jupyter 适合逐段运行、快速观察张量和实验结果。可以先用 Jupyter 验证单个操作，再把稳定代码整理到 Python 项目中。

## 二、数据管线

Dataset 定义“如何找到一个样本”，通常实现 __len__ 返回样本数，实现 __getitem__(index) 读取样本和标签。Dataset 不应承担完整训练逻辑，批量组织交给 DataLoader。

DataLoader 把 Dataset 包装成批次。batch_size 决定每次送入模型的样本数；训练集通常 shuffle，验证集通常不随机打乱；num_workers 提高数据读取并行度，但 Windows 环境需注意多进程启动方式。

Transform 把原始样本变成模型需要的形式，包括裁剪、缩放、翻转、张量转换和归一化。训练集可以加入合理的数据增强，验证集通常只做确定性的尺寸统一和归一化。典型数据流是：文件或图像 → Dataset 读取 → Transform 预处理 → DataLoader 组成 batch → 模型输入。

## 三、TensorBoard

TensorBoard 用于记录损失、准确率、图像样本、模型结构和训练曲线。它能帮助发现输入图像损坏、归一化错误、标签错位、训练不收敛和过拟合。可视化不是装饰，而是训练排错的一部分。

## 四、神经网络结构

自定义网络通常继承 nn.Module，在 __init__ 中声明层，在 forward 中定义数据流。阅读网络时必须确认输入形状、每层如何改变形状，以及最终输出维度为何匹配类别数。nn.Sequential 适合简单串联模型，存在分支或跳连时应显式编写 forward。

卷积层通过局部感受野和共享参数提取空间特征。重点检查输入输出通道、卷积核、步长、填充和输出尺寸。二维卷积输入通常包含批次、通道、高度、宽度四个维度。最大池化压缩空间尺寸并保留局部强响应；激活函数提供非线性；线性层常作为分类头。

图像分类模型常见结构是：卷积提取特征 → 激活 → 池化 → 重复堆叠 → 展平 → 线性层输出类别分数。卷积后接线性层前，要计算展平后的维度，否则容易产生形状不匹配。

## 五、训练闭环

损失函数把预测与标签的差异变成可优化的标量。要根据标签格式选择损失函数，并确认模型输出的形状和数值含义匹配。

一次训练迭代通常是：清空上一轮梯度，前向计算预测，计算损失，反向传播，优化器更新参数。对应的代码顺序是 optimizer.zero_grad、model(input)、criterion(output, target)、loss.backward、optimizer.step。忘记清空梯度会造成累积，忘记 backward 或 step 则不会完成正常更新。

优化器依据梯度更新参数。学习率过大可能导致损失震荡或发散，过小则收敛缓慢。训练异常时，先确认模型处于训练模式、参数允许求梯度，再检查学习率、输入和标签。

## 六、保存、加载、GPU 与验证

工程中通常保存模型参数或检查点。断点续训时还应保存优化器状态、当前 epoch 和验证指标。加载时确认模型结构与参数匹配，并处理 CPU/GPU 映射。

GPU 训练要求模型和数据位于同一设备：先选择 cuda 或 cpu，再迁移模型，并把每个 batch 的输入和标签迁移到同一设备。只迁移模型或只迁移数据都会产生设备不匹配或无法利用 GPU 的问题。

验证阶段不更新参数，应关闭梯度并切换到 eval 模式。Dropout、BatchNorm 等层在训练和评估阶段行为不同。验证指标要按真实样本数正确累计，不能忽略最后一个不完整批次。

## 七、可迁移的完整项目模板

1. 配置设备、随机种子和超参数。
2. 构造训练集、验证集和 DataLoader。
3. 定义模型并检查关键层的输入输出形状。
4. 选择损失函数和优化器。
5. 编写训练循环，记录损失和准确率。
6. 每个 epoch 后执行验证。
7. 保存最佳模型或训练检查点。
8. 用独立样本检查预测结果。

先让一个很小的数据集过拟合，是高效排错方法。如果连小数据都无法拟合，优先检查标签、损失、输出维度、梯度和预处理，而不是立刻增加网络层数。

## 常见坑点

- 把 CPU 环境中 CUDA 不可用误认为安装失败。
- 训练集和验证集使用了不一致的归一化。
- 标签从 1 开始编号，但分类损失要求从 0 开始。
- 卷积输出展平维度计算错误。
- 忘记 zero_grad 或 optimizer.step。
- 验证时忘记 eval 和关闭梯度。
- GPU 上的模型与 CPU 上的输入混用。
- 只看训练准确率，不看验证集，导致过拟合未被发现。
- 直接复制旧教程代码，却忽略当前 PyTorch 版本的 API 变化。
