# 硬件工程师入门教程：实训路线与验收标准

## 实训原则

每个项目都按“计算 -> 仿真/纸面分析 -> 面包板或模块验证 -> 测量 -> 记录 -> 改版”的顺序进行。上电前必须确认极性、最大电压、最大电流、功耗和可能的感性尖峰。

## 项目一：电阻、电容基础测量

对应 P1-P20。完成电阻识别、分压、限流、RC 充放电和上电延时。

验收：

- 实测分压值与计算值对比；
- 记录电阻误差和功耗；
- 测量 RC 时间常数；
- 改变负载后解释输出变化；
- 画出上电和断电波形。

## 项目二：二极管与整流滤波

对应 P29-P37。搭建半波、全波、峰值检波、钳位和稳压二极管电路。

验收：

- 记录正向压降；
- 测量整流后纹波；
- 检查稳压二极管限流和功耗；
- 给输入施加反向电压时确认器件耐压裕量；
- 为感性负载画出续流路径。

## 项目三：三极管与 MOS 开关

对应 P38-P65。分别用三极管、NMOS 和 PMOS 控制 LED、继电器或小电机。

验收：

- 控制端悬空时负载保持安全默认状态；
- 测量导通压降和负载电流；
- 验证低端与高端开关的差异；
- 检查 MOS 实际 `Vgs`，不能只看栅极对地电压；
- 用示波器观察栅极上升、下降和开关节点振铃；
- 感性负载必须有续流或钳位。

## 项目四：比较器、运放与电流采样

对应 P66-P92。完成阈值检测、迟滞比较器、同相/反相放大、低端采样和恒流源。

验收：

- 比较器输入范围和上拉电阻正确；
- 输入接近阈值时无明显颤振；
- 运放共模范围、输出摆幅和反馈稳定；
- 采样电阻功耗和压降满足需求；
- 高端采样验证共模电压；
- 恒流源在空载、正常负载和短路状态下均不会失控。

## 项目五：BUCK/BOOST 电源

对应 P93-P103。先分析现成模块，再尝试搭建低功率验证电路。

验收：

- 记录输入、输出、电流、效率和纹波；
- 检查电感饱和电流和峰值电流；
- 验证开关管、二极管和电容耐压；
- 示波器观察开关节点和电感电流；
- 检查启动、空载、满载、短路和输入突变；
- 确认高 di/dt 回路足够短。

## 项目六：纯硬件循迹小车

对应 P104-P115。分模块完成传感器、比较器、控制逻辑、电机驱动和电源。

验收顺序：

1. 单独测量黑线传感器输出；
2. 验证比较器阈值和迟滞；
3. 根据传感器组合列出真值表；
4. 不接电机时验证方向控制逻辑；
5. 低速接入电机并检查电源跌落；
6. 测试直线、弯道、不同光照和电池电压；
7. 测试堵转和急停是否导致逻辑复位。

## 项目七：PCB 实战

对应 P116-P125。把循迹小车电路转成可制造 PCB。

验收：

- 原理图引脚编号与数据表一致；
- 每个器件都有正确封装和方向标识；
- 电源、地和大电流网络完整；
- 功率回路、模拟采样和数字逻辑布局分区；
- 线宽、过孔、铜厚和间距满足电流与电压；
- 关键器件旁有去耦和测试点；
- 接插件、安装孔和板框满足机械结构；
- DRC、丝印、阻焊、钻孔和生产文件均检查。

## 故障排查顺序

1. 是否接反电源或超过器件耐压？
2. 电源空载和带载电压是否正常？
3. 地线和回流路径是否可靠？
4. 控制端是否悬空或逻辑电平不匹配？
5. 器件是否进入正确工作区？
6. 是否存在感性尖峰、寄生振荡或热失控？
7. 测量仪器接地是否改变了被测电路？
8. PCB 上封装、引脚、方向和网络是否一致？

## 最终作品建议

制作一块带传感器接口、电机驱动、电源管理和测试点的纯硬件循迹小车控制板。提交以下材料：需求说明、参数计算、原理图、BOM、PCB 文件、测试记录、故障记录和版本修改说明。这样才能把课程知识转化为可复用的硬件工程能力。
