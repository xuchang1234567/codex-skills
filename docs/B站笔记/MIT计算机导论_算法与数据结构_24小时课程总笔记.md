## 课程主线

整套课程可以分成六层：

1. 计算与算法的基本视角：如何描述问题、程序和资源消耗。
2. 线性数据结构与动态数组：从抽象接口理解操作成本。
3. 哈希、排序、树和堆：组织数据、查找和优先级处理。
4. 图搜索：BFS、DFS、连通性和搜索树。
5. 加权最短路：Dijkstra、Bellman-Ford 及其适用条件。
6. 动态规划与复杂度：把递归结构转化为可计算、可证明、可优化的算法。

课程中间穿插 Problem Session、Quiz Review 和 Course Review。它们不是附属内容，而是把“听懂概念”转化为“能独立建模和解题”的关键环节。

## 分 P 学习导航

### 1. Algorithms and Computation

入口课，建立算法和计算的基本语言：问题、输入输出、程序、正确性和运行成本。学习重点是学会把自然语言任务转成可执行步骤，并开始区分“能运行”和“运行得足够快”。建议练习：给一个简单查找任务，写出朴素算法并估算操作次数。

### 2. Data Structures and Dynamic Arrays

从数据结构抽象和动态数组出发，理解存储布局、索引、扩容、插入和删除的成本。不要只记住某个操作是 O(1) 还是 O(n)，要理解成本来自数据搬移、空间扩容和访问方式。练习时比较数组、链表和动态数组在不同操作上的取舍。

### 3. Hashing

哈希把对象映射到表位置，以较低平均成本完成查找、插入和删除。核心问题包括哈希函数、冲突、冲突处理和负载因子。应同时区分平均复杂度与最坏情况，理解为什么好的哈希表依赖合理的分布和扩容策略。

### 4. Problem Session 2

用题目检验前面的数据结构和复杂度分析。建议暂停视频独立完成题目，再比较讲解中的建模方式；不要只看最终答案，因为真正要训练的是识别数据结构、确定不变量和估算复杂度。

### 5. Linear Sorting

讨论不完全依赖比较的排序思路，例如利用键值范围、位数或桶结构获得线性或近线性表现。学习重点是算法的适用前提：键的类型、范围、位数和额外空间都会决定所谓“线性”是否成立。练习时写清楚输入假设，而不是只背算法名称。

### 6. Problem Session 3

该分 P 字幕缺失。应结合第 5 分 P 的线性排序、前面数据结构和复杂度内容补做题目，重点检查排序正确性、稳定性、空间成本和输入限制。

### 7-8. Binary Trees

二叉树部分从树的递归结构、遍历和基本操作展开。要掌握根、子树、叶、深度和高度等概念，理解前序、中序、后序遍历为何天然适合递归表达。树的形状直接影响操作成本，因此不能只看节点数量，还要关注是否退化成链。

### 9. Problem Session 4

围绕树结构进行练习，适合重点训练递归函数的边界条件：空树、叶节点、只有一侧子树以及返回值如何向上汇总。建议手动画出递归调用栈，再写迭代版本进行比较。

### 10. Binary Heaps

堆用于维护动态集合中的最小值或最大值，核心操作是插入、取顶和删除顶端元素。学习时抓住堆序性质、完全二叉树形状、向上调整和向下调整；不要把堆误认为“完全排序的数组”，堆只保证局部优先级关系。它是优先队列和 Dijkstra 的重要基础。

### 11. Breadth-First Search

BFS 使用队列按层扩展图，适用于无权图中的最短边数路径。必须掌握 visited 标记、父节点记录和搜索层次；若图用邻接表表示，复杂度通常与顶点数和边数相称。练习时输出一棵 BFS 搜索树，验证路径如何恢复。

### 12. Quiz 1 Review

这是对前半段知识的综合复习。建议按“抽象接口、正确性、复杂度、边界条件”四个维度整理错题，而不是只记录答案。重点回顾动态数组、哈希、排序、树、堆和 BFS 之间的适用场景。

### 13-14. Depth-First Search 与续讲

DFS 使用递归或显式栈深入探索图。关键不只是遍历顺序，还包括进入时间、离开时间、父子关系和搜索树结构。DFS 是连通性、环检测、拓扑相关问题和后续图算法的基础。要特别注意无向图和有向图中“已访问”的含义不同。

### 15. Weighted Shortest Paths

进入加权图最短路径。首先要明确边权是否非负、是否允许负权、需要单源还是多源结果。最短路算法不是万能模板，算法选择必须由图的权重条件和目标决定。

### 16. Problem Session 5

围绕图搜索和加权最短路训练算法建模。建议在做题时先写出图的表示方式、初始化状态、松弛操作和终止条件，再考虑代码。

### 17. Bellman-Ford

Bellman-Ford 通过反复松弛边处理单源最短路，能够识别负权边甚至负权环，但代价通常高于非负权图上的专用方法。学习重点是理解“经过 k 条边的最短路”这一层次结构，以及为什么重复若干轮松弛能够传播最短距离。

### 18. Problem Session 6

综合练习最短路和松弛思想。应能解释某次松弛为什么更新、为什么不更新，并能构造负权边或负权环的反例。

### 19. Dijkstra

Dijkstra 利用当前最小暂定距离节点逐步确定最短路，通常配合优先队列实现。它依赖非负边权；存在负权边时，已经确定的距离可能被后续路径改小，算法前提失效。需要掌握初始化、松弛、优先队列过期条目和路径恢复。

### 20. Problem Session 7 与 Quiz 2 Review

这部分用于检验最短路、堆和图搜索的组合应用。建议对比 BFS、Bellman-Ford 和 Dijkstra：无权图、可负权图、非负权图分别适合什么方法，复杂度和正确性依据是什么。

### 21-23. Dynamic Programming

动态规划解决具有重叠子问题和最优子结构的问题。核心流程是：定义状态，写出状态转移，确定初始条件和计算顺序，最后恢复答案。不要从“套模板”开始，而应先问：子问题是什么、状态是否足够表达未来决策、同一子问题是否会重复出现。

第 23 分 P 字幕缺失，动态规划后段需要结合视频画面或课程材料补核。其余动态规划分 P 应重点练习递归记忆化与自底向上的差异，以及时间、空间复杂度如何从状态数量和每个状态的转移数量得到。

### 24-25. Complexity

复杂度部分把前面的算法放回资源分析框架，讨论输入规模、渐进增长、时间/空间权衡和算法可扩展性。第 25 分 P 字幕缺失，建议重点补做复杂度比较和算法改进题，不要只记 O 记号；要说明每一层循环、递归分支、状态和边的来源。

### 26. Quiz 3 Review

围绕动态规划和复杂度进行综合复习。建议把每道题整理成五行：问题、状态、转移、边界、复杂度。若无法写出其中一项，说明算法还没有真正掌握。

### 27. Course Review

全课回顾，把数据结构、图算法和动态规划放在同一张选择表中。复习时应从问题特征反推算法，而不是从算法名称寻找题目：需要动态最小值时想到堆，需要无权最短路时想到 BFS，存在负权时检查 Bellman-Ford，非负权加权图时考虑 Dijkstra，有重叠子问题时考虑 DP。

### 28. Algorithms: Next Steps

课程收束到进一步学习方向：算法不是孤立技巧，而是建立在抽象、证明、复杂度和工程实现之间。后续学习可以继续深入摊销分析、随机化算法、高级图算法、字符串算法和更复杂的数据结构。

## 推荐学习法

每个正课分 P 采用四遍法：第一遍看问题背景，第二遍暂停写定义和伪代码，第三遍独立完成例题，第四遍整理复杂度和失败原因。Problem Session 与 Quiz Review 不要连续播放，先独立做题再回看。

每学完一个主题，至少留下三类成果：一个可运行实现、一个反例、一个复杂度解释。例如实现 Dijkstra 时，同时写出负权边反例；实现哈希表时，测试冲突和扩容；实现 DP 时，比较记忆化和自底向上。
