## 一句话总论

机器人软件不是单个算法，而是一条分层系统：底层 Linux/设备通信提供运行环境，ROS2 用消息和节点组织系统，URDF/TF 描述机器人几何关系，`ros2_control` 连接控制器，MoveIt 2 负责机械臂规划，Nav2 负责移动机器人导航，Actions 则承载可反馈、可取消的长时任务。

## 核心概念卡

### 1. ROS2 节点与通信

节点是一个职责相对单一的进程或组件。话题适合连续数据流，例如传感器、里程计和速度指令；服务适合快速请求-响应；Action 适合持续执行并需要反馈、取消或结果的任务。设计系统时先画出节点图，再决定通信类型，避免把长任务塞进同步服务。

### 2. URDF、link、joint 与 TF

URDF 用 `link` 表示刚体，用 `joint` 描述父子 link 的关系。每个子 link 只能有一个父 link，但一个父 link 可以连接多个子 link，整体形成树。关节的 `origin`、旋转轴和类型决定 TF 变换；位置错、旋转轴错或父子关系错，都会导致 RViz 中模型漂移或轮子旋转异常。[1]

常见流程：编写 URDF → 创建 ROS2 package → 用 launch 启动 `robot_state_publisher` → 发布关节状态 → 在 RViz 检查坐标轴、模型姿态和 TF 树。

### 3. ros2_control

`ros2_control` 把硬件读写和控制器逻辑分开。硬件接口提供状态和命令接口，controller manager 负责加载、激活和调度控制器。先在仿真中验证接口名称、关节名、单位和更新频率，再接真实硬件，可显著降低调试成本。

### 4. MoveIt 2

MoveIt 2 面向机械臂运动规划：输入机器人模型、关节限制、规划组和目标姿态，输出无碰撞的关节轨迹，再交给控制器执行。规划失败时应分别检查 TF、碰撞模型、关节范围、规划组配置和控制器连接，不要只反复调规划时间。

### 5. Nav2

Nav2 是移动机器人导航栈，典型链路包括传感器输入、TF、定位/地图、代价地图、全局规划、局部控制和行为树。导航问题通常不是单一算法故障，而是坐标系、时间戳、里程计、激光数据或参数配置不一致造成的。

### 6. ROS2 Actions

Action 由 goal、feedback、result 组成，客户端可以等待结果、持续查看反馈并取消任务。导航、机械臂运动和长时间路径跟踪都适合 Action。调试时分别确认 action 名称、接口类型、服务器是否 active、goal 是否接受以及反馈频率。

## 实践路线

1. 在 Ubuntu/虚拟机中熟悉终端、包管理、环境变量和进程。
2. 用 Python 写一个最小发布者/订阅者，再用 C++ 重写关键节点。
3. 用 Arduino 采集传感器或控制电机，通过串口把数据发送到树莓派。
4. 建立 ROS2 workspace，创建 package，编译并 source 环境。
5. 先用话题传输传感器和速度数据，再用 service 做短请求，用 Action 做长任务。
6. 编写最小 URDF，只放 base、一个传感器和一个轮子；在 RViz 中逐步校准 `origin` 和旋转轴。
7. 在仿真中接入 `ros2_control`，确认关节状态和命令接口，再接控制器。
8. 对机械臂使用 MoveIt 2，对移动机器人使用 Nav2；先验证 TF 和传感器，再调规划参数。
9. 为每个实验保留 launch、参数 YAML、URDF、日志和复现命令。

## 排错清单

- `ros2` 命令找不到：检查 ROS2 环境是否 source，workspace overlay 顺序是否正确。
- 节点能启动但收不到数据：核对 topic 名称、QoS、消息类型和命名空间。
- RViz 模型错位：检查 URDF 父子关系、joint origin、轴方向和 TF 发布者。
- 控制器无法激活：检查关节名称、command/state interface、控制器配置和硬件状态。
- MoveIt 规划失败：检查规划组、关节边界、碰撞几何和目标姿态坐标系。
- Nav2 原地打转或无法定位：检查地图/里程计/激光的 TF、时间戳和坐标约定。
- Action 无反馈：确认服务器已启动、接口类型一致，并检查 goal 是否真的被接受。

## 常见误区

- 先调高级算法、后检查 TF 和数据流，通常会浪费大量时间。
- 把 topic、service、Action 当成同一种通信方式，导致任务模型不匹配。
- 只在实体机器人上调试，不先用 Gazebo/RViz 验证模型和接口。
- URDF 中几何尺寸正确但 joint origin 错误，模型仍然会表现异常。
- ROS2 包能编译不代表运行环境正确，必须同时检查 source、参数和运行时依赖。
